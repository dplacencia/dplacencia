## i18n quick notes

- Translations live in `src/i18n/en.ts` and `src/i18n/es.ts`. Shape is defined by `src/i18n/index.ts`.
- Pages pull the correct dictionary with `getTranslations(locale)` and pass `t` + `locale` into `BaseLayout` and the page components.
- Language-prefixed routes:
  - English (default): `/`, `/services`, `/about`, `/contact`, `/working-together`
  - Spanish: `/es`, `/es/services`, `/es/about`, `/es/contact`, `/es/working-together`
- Language switcher builds the counterpart URL by adding/removing the `/es` prefix while preserving the path.
- To add new strings:
  1. Add the key to both `en.ts` and `es.ts`.
  2. If needed, extend the type in `src/i18n/index.ts` (it derives from `en`).
- To add a new language:
  1. Create `src/i18n/<lang>.ts`.
  2. Extend `Locale` and `translations` in `src/i18n/index.ts`.
  3. Add `<lang>` route files mirroring the existing ones, passing the new `t`.
